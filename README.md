# 상품 소싱 작업대

**AI 재테크 연구소** · [네이버 카페](https://cafe.naver.com/aiprogram1)

네이버 공식 API로 **팔 만한 자리를 찾아** 주는 도구입니다.
키워드를 몰라도 분야만 고르면 됩니다. **위탁**과 **도매**를 나눠서 봅니다.

---

## 무엇을 하나

| | |
|---|---|
| **찾기** | 분야만 고르면 300번쯤 훑어 최대 60개를 찾아옵니다 |
| **가르기** | 경쟁 실체 · 브랜드 장악도 · 제목 획일성 · 가격 폭으로 걸러냅니다 |
| **계산** | 떼오는 값 상한 · 마진 · **배송대행비** · **관세·부가세**까지 |
| **올리기** | 상품명 3벌 · 상세페이지 8줄 · 찍어야 할 사진 목록 |
| **지키기** | 골든타임(수요↑ 셀러↔) · 치킨게임(최저가 붕괴) · 적중률 |

## 이 도구가 **못 하는 것** (먼저 읽으세요)

정직하게 밝힙니다. 이걸 모르고 쓰면 손해가 납니다.

- **네이버 웹 화면과 최저가가 다릅니다.** 웹은 광고를 끼워 넣고 검색 API와
  색인이 다릅니다. **구조적으로 같아질 수 없습니다.** 그래서 카드마다
  `[이 가격 상품 보기]`를 두었습니다 — 그 값이 나온 **바로 그 상품**으로 갑니다.
- **최저가는 내 물건과 같은 제품이 아닐 수 있습니다.** 그 키워드로 검색했을 때
  가장 싸게 걸리는 물건의 값입니다. `가격비교로 묶인 같은 제품`이라고 뜨면
  네이버가 동일 제품으로 확인해준 것이라 더 정확합니다.
- **판매량·리뷰 수를 알 수 없습니다.** 네이버 API가 주지 않습니다.
  '수요'는 **검색량**이지 구매량이 아닙니다.
- **관세율은 통념입니다.** 품목(HS코드)마다 다릅니다. 발주 전 관세청·관세사에
  확인하세요. 도구는 "세금이 이만큼 붙을 수 있다"를 알려줄 뿐입니다.
- **기준선(15%·30%·경쟁 수)은 임의로 정한 값입니다.** 검증된 공식이 아닙니다.
- **상위노출을 보장하지 않습니다.** 노출은 판매·리뷰·클릭률·광고가 함께 정합니다.
- **크롤링하지 않습니다.** 쿠팡·11번가·다나와는 공개 검색 API가 없어
  자동 비교가 안 됩니다. 링크로 열어드릴 뿐입니다.

## 시작하기

### 1. 네이버 열쇠 발급 (필수)

1. [developers.naver.com](https://developers.naver.com) 로그인
2. **애플리케이션 등록** → **검색**과 **데이터랩(쇼핑인사이트)** 선택
3. `Client ID` / `Client Secret` 복사

> 열쇠는 **각자 브라우저에만** 저장됩니다. 서버로 보내지 않고 보관하지 않습니다.
> 그래서 여러 사람이 써도 한도를 나눠 쓰지 않습니다.

### 2. 내 컴퓨터에서 실행

```bash
git clone https://github.com/<your-id>/sourcing-workbench.git
cd sourcing-workbench
pip install -r requirements.txt
python run_web.py
```

Windows는 **`run_web.bat` 더블클릭**이면 됩니다.
빈 포트를 알아서 찾고 브라우저를 열어줍니다.

### 3. Render에 올리기

1. 이 저장소를 GitHub에 올립니다
2. Render → **New** → **Blueprint** → 저장소 선택
3. `render.yaml`을 알아서 읽습니다

---

## ⚠️ Render 무료 플랜에서 알아야 할 것

- **자료가 날아갑니다.** 재배포·재시작마다 `market_history.sqlite`가
  초기화됩니다 → **관심목록·적중률·키워드풀이 사라집니다.**
  계속 쌓으려면 `render.yaml`의 `disk` 주석을 풀고 유료 플랜으로 바꾸세요.
- **15분 접속이 없으면 잠듭니다.** 다음 접속이 30초쯤 느립니다.
- **한 요청이 40초쯤 걸립니다.** 네이버 호출 간격 때문이며 줄일 수 없습니다.

---

## 여럿이 쓸 때

- **관심 목록은 브라우저마다 따로**입니다 (`owner` 로 나눕니다).
- **시장 정보(골든타임·적중률·키워드풀)는 공유**합니다 — 여럿이 볼수록
  정확해지기 때문입니다. 이건 의도된 설계입니다.
- **네이버 열쇠는 각자 것**을 씁니다. 한도를 나눠 쓰지 않습니다.

## 구조

```
run_web.py            실행기 (빈 포트 자동)
selftest.py           내보내기 전 검사 — 반드시 통과해야 함
web/
  server.py           FastAPI
  static/index.html   화면 (단일 파일)
discovery/
  auto_scan.py        발굴 엔진 (3단계 병렬)
  title_mining.py     제목 채굴 — 상위 노출 제목에서 시장의 언어를 캠
  competition.py      경쟁 실체 (링크로 카탈로그 판별)
  market_price.py     시장 가격
  quality.py          좋은 상품인가 (브랜드 장악·제목 획일성)
  mode.py             위탁 vs 도매 (기준이 다름)
  customs.py          관세·부가세
  suppliers.py        떼올 곳
  checklist.py        시작 전 준비물
  listing.py          상품명·상세페이지
  season.py           시즌 선행
  tracker.py          골든타임·적중률·관심목록
```

## 검사

```bash
python selftest.py
```

`compile` · `import` · **화면 검사**(죽은 코드, 리스너 사고, 필수 요소)를 봅니다.
`ast.parse`가 "인자 중복"을 못 잡아 사용자 PC에서 터진 적이 있어
`compile()`로 바꿨습니다.

## 규칙

- 네이버 **공식 API만** 씁니다. 크롤링하지 않습니다.
- 추측한 숫자는 **추측이라고 밝힙니다.**
